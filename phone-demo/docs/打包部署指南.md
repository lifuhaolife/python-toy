## 2. 方式一：Buildozer 打包 (Linux/Mac)

### 2.1 环境要求

- Ubuntu 18.04+ / Debian 10+ / macOS 10.15+
- Python 3.8-3.11
- 磁盘空间：> 10GB
- 内存：> 4GB (推荐 8GB)

### 2.2 安装依赖

**Ubuntu/Debian**:
```bash
sudo apt update
sudo apt install -y \
    git ffmpeg libffi-dev libssl-dev \
    python3 python3-pip python3-venv \
    python3-dev build-essential \
    autoconf automake libtool \
    libltdl-dev libffi-dev \
    libgstreamer1.0-dev gstreamer1.0-plugins-base \
    gstreamer1.0-plugins-good \
    libgl1-mesa-dev libgles2-mesa-dev \
    openjdk-11-jdk cmake
```

**macOS**:
```bash
brew install \
    git ffmpeg libffi openssl \
    python3 autoconf automake \
    libtool pkg-config \
    openjdk@11 cmake
```

### 2.3 安装 Buildozer

```bash
cd /toy/phone-demo/app

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装 buildozer
pip install buildozer cython

# 或使用官方推荐版本
pip install "buildozer>=1.5.0"
```

### 2.4 初始化配置

```bash
# 初始化 (首次运行会创建 buildozer.spec)
buildozer init

# 如果已有 buildozer.spec，跳过此步
```

### 2.5 配置 buildozer.spec

编辑 `buildozer.spec` 文件，确保以下配置正确：

```ini
[app]
title = 儿童智能语音玩具
package.name = toyphone
package.domain = org.toy

source.dir = .
source.include_exts = py,png,jpg,kv,atlas
version = 1.0.0

requirements = python3,kivy,aiohttp,pyaudio
orientation = portrait

osx.python_version = 3
osx.kivy_version = 2.3.0

fullscreen = 0
android.permissions = INTERNET,RECORD_AUDIO,MODIFY_AUDIO_SETTINGS
android.minapi = 21
android.sdk = 24
android.api = 30
android.ndk = 25b
android.arch = arm64-v8a
android.accept_sdk_license = True

android.add_aars =
android.add_jars =

p4a.source_dir =
p4a.bootstrap = sdl2
p4a.extra_args = --blacklist-requirements=libffi

[buildozer]
log_level = 2
warn_on_root = 1
```

### 2.6 开始打包

```bash
# 打包 debug 版本 (用于测试)
buildozer -v android debug

# 打包 release 版本 (用于发布)
buildozer -v android release
```

### 2.7 打包输出

打包完成后，APK 文件位置：

```
bin/toyphone-1.0.0-debug.apk    # Debug 版本
bin/toyphone-1.0.0-release.apk  # Release 版本
```

### 2.8 首次打包时间

- 首次打包：**60-90 分钟** (下载 Android SDK/NDK)
- 后续打包：**10-20 分钟**

## 3. 方式二：WSL2 打包 (Windows)

### 3.1 安装 WSL2

**Windows 10/11 管理员 PowerShell**:
```powershell
# 启用 WSL
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

# 启用虚拟机平台
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

# 重启电脑后，设置 WSL2 为默认
wsl --set-default-version 2
```

### 3.2 安装 Ubuntu

**Microsoft Store**:
1. 打开 Microsoft Store
2. 搜索 "Ubuntu 22.04"
3. 安装并启动
4. 设置用户名和密码

### 3.3 WSL2 环境配置

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装依赖
sudo apt install -y \
    git ffmpeg libffi-dev libssl-dev \
    python3 python3-pip python3-venv \
    build-essential autoconf automake \
    libtool libltdl-dev \
    libgstreamer1.0-dev \
    gstreamer1.0-plugins-base \
    gstreamer1.0-plugins-good \
    openjdk-11-jdk cmake \
    wget curl

# 设置 Java 环境
sudo update-alternatives --config java
```

### 3.4 访问 Windows 文件

WSL2 中访问 Windows 文件：
```bash
cd /mnt/c/Users/lenovo/Claudes/toy/phone-demo/app
```

### 3.5 安装 Buildozer

```bash
# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装 buildozer
pip install buildozer cython
```

### 3.6 打包

```bash
# 确保在正确的目录
cd /mnt/c/Users/lenovo/Claudes/toy/phone-demo/app

# 开始打包
buildozer -v android debug
```

### 3.7 获取 APK

打包完成后，APK 文件在：
```
bin/toyphone-1.0.0-debug.apk
```

可以通过 Windows 资源管理器访问：
```
\\wsl$\Ubuntu\mnt\c\Users\lenovo\Claudes\toy\phone-demo\app\bin\
```

## 4. 方式三：直接安装测试 (快速)

### 4.1 使用 Pydroid 3 (推荐快速测试)

**步骤**:

1. **手机安装 Pydroid 3**
   - Google Play 搜索 "Pydroid 3"
   - 或从 https://pydroid.net 下载

2. **传输代码到手机**
   ```
   方式 A: USB 传输
   手机存储/Download/toyphone/app/
   
   方式 B: 二维码
   电脑启动 HTTP 服务器:
   cd phone-demo
   python -m http.server 8080
   
   手机浏览器访问下载
   ```

3. **Pydroid 3 中安装依赖**
   - 打开 Pydroid 3
   - 菜单 → Pip
   - 输入：`kivy aiohttp`
   - 点击 Install

4. **运行 APP**
   - 打开 Pydroid 3
   - 浏览到 `main.py`
   - 点击运行按钮

### 4.2 使用 Termux (高级)

```bash
# 安装 Termux (从 F-Droid)
# https://f-droid.org/packages/com.termux/

# 更新
pkg update && pkg upgrade

# 安装 Python
pkg install python

# 安装依赖
pkg install portaudio pyaudio

# 安装 Python 包
pip install kivy aiohttp

# 运行
cd /sdcard/Download/toyphone/app
python main.py
```

## 5. APK 安装到手机

### 5.1 传输 APK 到手机

**方式 A: USB 传输**
```
1. USB 连接手机
2. 复制 APK 到手机存储
3. 手机文件管理器找到 APK
4. 点击安装
```

**方式 B: 局域网传输**
```bash
# 电脑启动 HTTP 服务器
cd phone-demo/app/bin
python -m http.server 8080

# 手机浏览器访问
http://电脑 IP:8080/toyphone-1.0.0-debug.apk
# 下载并安装
```

**方式 C: 二维码下载**
```bash
# 生成下载链接二维码
# 使用 https://qrcode.kaywa.com/
# 输入下载链接，生成二维码
# 手机扫码下载
```

### 5.2 允许未知来源

**Android 8.0+**:
```
设置 → 应用和通知 → 高级 → 特殊应用权限 → 安装未知应用
→ 选择 Chrome/文件管理器 → 允许
```

**Android 7.0 及以下**:
```
设置 → 安全 → 未知来源 → 允许
```

### 5.3 安装 APK

1. 打开文件管理器
2. 找到下载的 APK 文件
3. 点击安装
4. 等待安装完成
5. 打开应用

## 6. 常见问题

### 6.1 Buildozer 相关

**Q1: `buildozer: command not found`**
```bash
# 确保在虚拟环境中
source venv/bin/activate

# 或重新安装
pip install --user buildozer
```

**Q2: Java 环境错误**
```bash
# 检查 Java
java -version

# 设置 JAVA_HOME
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64
```

**Q3: SDK 下载失败**
```bash
# 使用国内镜像
# 编辑 ~/.buildozer/android/platform/android-sdk/tools/bin/sdkmanager
# 或手动下载 SDK 放到 ~/.buildozer/android/platform/android-sdk/
```

**Q4: NDK 版本不匹配**
```bash
# 在 buildozer.spec 中指定
android.ndk = 25b

# 清理重新下载
buildozer android clean
buildozer android debug
```

### 6.2 运行时错误

**Q1: 无法连接服务器**
```
检查:
1. 服务器是否启动
2. 手机和电脑是否同一 WiFi
3. 防火墙是否开放
4. IP 地址是否正确
```

**Q2: 麦克风权限被拒绝**
```
设置 → 应用 → 儿童智能语音玩具 → 权限 → 麦克风 → 允许
```

**Q3: APP 闪退**
```
1. 查看 logcat 日志
2. 检查 Python 依赖是否完整
3. 重新打包
```

### 6.3 打包优化

**减小 APK 体积**:
```ini
# buildozer.spec
[app]
# 只包含需要的模块
p4a.extra_args = --blacklist-requirements=libffi \
                 --exclude-recipes=libffi
```

**加速打包**:
```bash
# 使用缓存
buildozer android debug --cached

# 使用国内镜像
export PIP_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple
```

## 7. 一键打包脚本

### 7.1 Linux/Mac 脚本

创建 `build.sh`:
```bash
#!/bin/bash

echo "🔧 开始打包儿童智能语音玩具..."

# 检查虚拟环境
if [ ! -d "venv" ]; then
    echo "创建虚拟环境..."
    python3 -m venv venv
fi

# 激活虚拟环境
source venv/bin/activate

# 安装依赖
echo "📦 安装依赖..."
pip install -q buildozer cython

# 检查 buildozer.spec
if [ ! -f "buildozer.spec" ]; then
    echo "❌ buildozer.spec 不存在!"
    exit 1
fi

# 开始打包
echo "🚀 开始打包..."
buildozer -v android debug

# 检查输出
if [ -f "bin/toyphone-1.0.0-debug.apk" ]; then
    echo "✅ 打包成功!"
    echo "📱 APK 位置：bin/toyphone-1.0.0-debug.apk"
else
    echo "❌ 打包失败!"
    exit 1
fi
```

使用:
```bash
chmod +x build.sh
./build.sh
```

### 7.2 Windows 批处理脚本

创建 `build.bat`:
```batch
@echo off
echo 🔧 开始打包儿童智能语音玩具...

REM 检查 WSL
wsl --version >nul 2>&1
if %errorlevel% neq 0 (
    echo ❌ WSL 未安装，请先安装 WSL2
    exit /b 1
)

REM 在 WSL 中执行打包
echo 🚀 在 WSL 中执行打包...
wsl bash -c "cd /mnt/c/%CD:~3%\ && ./build.sh"

echo 完成!
pause
```
