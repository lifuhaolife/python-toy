## 📊 日志标识说明

### APP 端标识

| 标识 | 说明 |
|------|------|
| `============================================================` | 重要操作开始/结束 |
| `------------------------------------------------------------` | 一般操作开始 |
| `✅` | 成功操作 |
| `❌` | 失败操作 |

### 服务器端标识

| 标识 | 说明 |
|------|------|
| `================================================================================` | 连接开始/结束 |
| `============================================================` | 请求处理开始/结束 |
| `------------------------------------------------------------` | 消息接收 |
| `✅` | 成功操作 |
| `❌` | 失败操作 |
| `⚠️` | 警告信息 |

```bash

```

```bash

```

## 💡 排查问题技巧

### 1. 连接问题

查看 APP 日志中的连接部分：
```
如果看到"连接失败"，检查：
- 服务器是否启动
- 服务器地址是否正确
- 防火墙是否阻止
```

### 2. 发送问题

查看 APP 日志中的发送部分：
```
如果看到"发送失败"，检查：
- WebSocket 是否连接
- 音频数据是否正确
```

### 3. 响应问题

查看服务器日志：
```
如果看到"AI 对话失败"，检查：
- AI_API_KEY 是否配置
- 网络连接是否正常
```

### 4. TTS 问题

查看服务器日志中的 TTS 部分：
```
如果看到"合成失败"，检查：
- edge-tts 是否正常
- 网络连接是否正常
```
