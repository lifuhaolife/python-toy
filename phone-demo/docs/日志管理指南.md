## 📝 日志文件命名规则

### 服务器日志
```
server_YYYYMMDD_HHMMSS.log
示例：server_20260224_143022.log
```

### APP 日志
```
app_YYYYMMDD_HHMMSS.log
示例：app_20260224_143022.log
```

**说明**:
- `YYYYMMDD`: 年月日
- `HHMMSS`: 时分秒（启动时间）

**位置**: `server/server.py`

```python
# 日志文件夹
LOG_DIR = os.path.join(os.path.dirname(__file__), 'logs')

# 日志文件命名
log_filename = datetime.now().strftime('server_%Y%m%d_%H%M%S.log')
```

## 🧹 日志清理

### 手动清理

**Windows 批处理**:
```batch
@echo off
echo 清理 7 天前的日志文件...
forfiles /p "server\logs" /s /m *.log /d -7 /c "cmd /c del @path"
forfiles /p "app\logs" /s /m *.log /d -7 /c "cmd /c del @path"
echo 清理完成！
pause
```

### 自动清理（Python 脚本）

```python
# clean_logs.py
import os
import time

def clean_old_logs(log_dir, days=7):
    """清理指定天数之前的日志"""
    now = time.time()
    cutoff = now - (days * 86400)  # 7 天前的时间戳
    
    for filename in os.listdir(log_dir):
        if filename.endswith('.log'):
            filepath = os.path.join(log_dir, filename)
            mtime = os.path.getmtime(filepath)
            
            if mtime < cutoff:
                os.remove(filepath)
                print(f"已删除：{filename}")

# 使用
clean_old_logs('server/logs', days=7)
clean_old_logs('app/logs', days=7)
```

## 🔧 日志配置选项

### 修改日志级别

编辑 `server/server.py` 或 `app/main.py`:

```python
# 可选级别：DEBUG, INFO, WARNING, ERROR, CRITICAL
logger.setLevel(logging.DEBUG)  # 更详细的日志
```

### 修改日志保留时间

```python
# 保留 7 天的日志（每小时一个文件，共 24*7=168 个文件）
file_handler = TimedRotatingFileHandler(
    log_filepath,
    when='H',
    interval=1,
    backupCount=168  # 改为 168
)
```

### 按天分割日志

```python
# 按天分割（而不是按小时）
file_handler = TimedRotatingFileHandler(
    log_filepath,
    when='D',        # 按天
    interval=1,
    backupCount=30   # 保留 30 天
)
```

## 📋 日志检查清单

每次测试后检查：

- [ ] 日志文件是否正常生成
- [ ] 日志文件名是否包含时间戳
- [ ] 日志内容是否包含关键信息
- [ ] 日志文件大小是否合理（< 10MB/小时）
- [ ] 旧日志是否定期清理

## 🆘 故障排查

### 问题：日志文件未生成

**检查**:
1. 日志文件夹是否存在
2. 是否有写入权限
3. 磁盘空间是否充足

### 问题：日志内容不完整

**解决**:
1. 检查日志级别设置
2. 确认 handler 已正确添加
3. 查看是否有异常中断

### 问题：日志文件过大

**解决**:
1. 减小 backupCount
2. 缩短轮转间隔
3. 定期清理旧日志
