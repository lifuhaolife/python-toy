## 📋 日志系统概述

项目使用 Python 标准 `logging` 模块，提供结构化日志输出。

---

## 🔧 日志配置

### 服务器端日志

**位置**: `server/server.py`

**配置**:
```python
logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    datefmt='%Y-%m-%d %H:%M:%S'
)
logger = logging.getLogger("toy-server")
```

**日志级别**:
- `INFO`: 一般信息
- `WARNING`: 警告信息
- `ERROR`: 错误信息
- `DEBUG`: 调试信息（需修改配置）

## 🖥️ 查看日志方法

### 方式 1：直接运行（推荐开发用）

```bash
cd server
python server.py
```

日志直接输出到控制台。

### 方式 2：后台运行 + 日志文件

```bash
# Windows
start /B python server.py > server.log 2>&1

# 实时查看日志
tail -f server.log
```

### 方式 3：使用监控脚本

```bash
cd server
monitor.bat
```

## 📱 APP 端日志

### 日志输出位置

1. **UI 日志区域**: APP 界面底部
2. **控制台输出**: 启动 APP 的命令行窗口

### 日志格式

```
[APP LOG] [HH:MM:SS] 消息内容
[APP ERROR] 错误详情
```

### 启用详细日志

编辑 `app/main.py`:
```python
# 添加调试日志
print(f"[DEBUG] 详细调试信息")
```

## 🔍 日志分析技巧

### 搜索特定设备日志

```bash
# Windows
findstr /C:"phone_001" server.log

# Linux/Mac
grep "phone_001" server.log
```

### 搜索错误日志

```bash
# Windows
findstr /C:"ERROR" server.log

# Linux/Mac
grep "ERROR" server.log
```

### 统计消息数量

```bash
# 统计连接数
findstr /C:"[WebSocket] 新连接" server.log | wc -l

# 统计识别次数
findstr /C:"[ASR] 识别结果" server.log | wc -l
```

## 🛠️ 调试技巧

### 启用 DEBUG 级别日志

修改 `server/server.py`:
```python
logging.basicConfig(
    level=logging.DEBUG,  # 改为 DEBUG
    ...
)
```

### 添加自定义日志

```python
logger.debug("调试信息")
logger.info("一般信息")
logger.warning("警告信息")
logger.error("错误信息")
```

### 捕获异常并记录

```python
try:
    # 可能出错的代码
    pass
except Exception as e:
    logger.error(f"发生错误：{e}", exc_info=True)
```

## 🆘 故障排查

### 没有日志输出

**检查**:
1. 日志级别是否正确
2. 服务器是否正常运行
3. 控制台是否被重定向

### 日志乱码

**解决**:
```bash
# Windows 设置 UTF-8
set PYTHONUTF8=1
python -u server.py
```

### 日志文件过大

**解决**:
```python
# 使用 RotatingFileHandler
from logging.handlers import RotatingFileHandler

handler = RotatingFileHandler(
    'server.log',
    maxBytes=10*1024*1024,  # 10MB
    backupCount=5
)
```
